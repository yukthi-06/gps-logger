# Implementation Plan for Offline Map Download Progress

**User Prompt:**
> show download status until download completes

1. **Investigate the Problem**:
    * The current implementation starts the MapLibre offline download in the background and only notifies the user when it's done via a Toast.
    * Users have no visibility into the progress or how many tiles are being downloaded.
    * We can extract real-time download metrics from `OfflineRegionStatus` inside the `OfflineRegionObserver.onStatusChanged` callback.

2. **Implementation Steps**:
    * **`activity_map.xml`**:
        * Add a `LinearLayout` at the bottom of the map containing a `TextView` (for status text) and a `ProgressBar` (for percentage).
        * Set this layout's visibility to `gone` by default.
    * **`MapActivity.java`**:
        * Bind the new UI components.
        * When the user clicks "Download Offline", set the progress layout to `VISIBLE`.
        * Inside `onStatusChanged`:
            * Check if `status.isRequiredResourceCountPrecise()` is true to calculate a real percentage.
            * Update the `ProgressBar` max to 100, and current progress based on `(completed / required) * 100`.
            * Update the `TextView` to show something like `"Downloading... 45 / 120 tiles (37.5%)"`.
            * Once `status.isComplete()` is true, hide the layout and show the success Toast.
        * Handle errors by hiding the layout and showing an error Toast.
