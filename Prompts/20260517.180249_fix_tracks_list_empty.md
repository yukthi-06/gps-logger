# Implementation Plan for Fixing Empty Tracks List

**User Prompt:**
> 'Tracks List' screen is empty. It should pick up files from the data folder at /sdcard/Vypeensoft/GPS_Location_Logger/

1. **Investigate the Problem**:
    * The `TracksListActivity` is using `Environment.getExternalStorageDirectory()` and `"Vypeensoft/GPS_Location_Logger"`. This resolves to `/storage/emulated/0/Vypeensoft/GPS_Location_Logger`.
    * If this is empty, it could be due to:
        a) Missing permissions (e.g., `MANAGE_EXTERNAL_STORAGE` or `READ_EXTERNAL_STORAGE`) in `TracksListActivity`. It might need to request them or at least check them. However, `MainActivity` already requests them.
        b) The folder path might not be resolving correctly. We should log the exact path it's trying to read, or hardcode `/sdcard/Vypeensoft/GPS_Location_Logger/` if the user explicitly wants `/sdcard/`. Usually `/sdcard/` is a symlink to `/storage/emulated/0/`, but let's be sure.
        c) `listFiles()` returning `null` because of scoped storage restrictions if permissions aren't properly maintained or the app hasn't been granted `MANAGE_ALL_FILES`.
2. **Implementation Steps**:
    * Update `TracksListActivity.java` to explicitly check and request `MANAGE_EXTERNAL_STORAGE` or `READ_EXTERNAL_STORAGE` if not granted.
    * Add error logging or Toast messages to show *why* the list is empty (e.g., "Directory does not exist", "Permission denied", "Folder is empty").
    * Ensure the path used is robust. Let's explicitly log the resolved path `dir.getAbsolutePath()`.
