# User Prompt
On app first use, ask for disk and Location permissions. Dont wait for any user activity to ask for permission

# Implementation Plan - Startup Permissions Request

We will check and prompt for both location and external storage (disk) permissions as soon as the application is launched, rather than waiting for a user action like clicking "Start Tracking" or "Record Point".

## Solution Design

### 1. Separate Startup Request from Tracking Action
If permissions are granted immediately upon app startup, we must **not** automatically trigger tracking or single point logging.
- We will add a boolean flag `isStartupPermissionRequest` in `MainActivity.java`.
- When the activity is launched (`onCreate`), we will set `isStartupPermissionRequest = true` and call our startup permission check.
- When manual actions are clicked ("Start Tracking" or "Record Point"), they will set `isStartupPermissionRequest = false` before verifying permissions.

### 2. Startup Permission Request Logic
- Implement `requestPermissionsOnStartup()` in `MainActivity.java` to check and request:
  - `MANAGE_EXTERNAL_STORAGE` on Android 11+ (API 30+)
  - `ACCESS_FINE_LOCATION`
  - `WRITE_EXTERNAL_STORAGE` on Android < 11 (API < 30)
  - `POST_NOTIFICATIONS` on Android 13+ (API 33+)
- This will match the permission definitions exactly as they are currently checked.

### 3. Handle Permission Results
- Modify `onRequestPermissionsResult` to check if `isStartupPermissionRequest` is true. If it is, log the approval/denial and do **not** trigger any tracking or recording operations.

## Proposed Changes

### 1. Update `MainActivity.java`
- Declare `private boolean isStartupPermissionRequest = false;`.
- At the end of `onCreate()`, invoke `requestPermissionsOnStartup()`.
- Add `requestPermissionsOnStartup()` method.
- Update `onRequestPermissionsResult(...)` to handle `isStartupPermissionRequest` flag.
