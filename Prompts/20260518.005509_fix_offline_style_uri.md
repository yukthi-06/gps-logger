# Implementation Plan for Fixing Offline 0/1 Tile Download Stuck

**User Prompt:**
> stuck at 'Downloading ... 0/1 tiles(0.0%)'

1. **Investigate the Problem**:
    * The progress being stuck at `0/1` tiles indicates that the `OfflineManager`'s background download thread is attempting to download the **style file** itself (the 1 initial resource) but getting stuck or failing.
    * This occurs because `asset://` URIs are handled via the Android `AssetManager` wrapper, which relies on JNI. MapLibre's background download threads running in C++ do not have standard, persistent access to the Android AssetManager JNI context. Thus, it cannot load `asset://osm_style.json` in the background, causing the tile list expansion to fail and get stuck at `0/1` forever.
    * **The Solution**: We will write the style JSON file dynamically to the app's internal files directory (`context.getFilesDir()`) and load it via a `file://` URI. Standard `file://` access uses direct C++ file I/O, which does not depend on JNI context and is fully supported on any thread by MapLibre's native offline engine.

2. **Implementation Steps**:
    * **`MapActivity.java`**:
        * In `onCreate()`, write the OpenStreetMap style JSON to a local file in `getFilesDir()`, e.g., `getFilesDir() + "/osm_style.json"`.
        * Save this file URI as `file:///...`.
        * Update the MapView style setup to load from this `file://` URI.
        * Update the `OfflineTilePyramidRegionDefinition` to use this `file://` URI.
