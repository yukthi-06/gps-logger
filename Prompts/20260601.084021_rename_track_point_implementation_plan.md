# Implementation Plan - Rename Track/Point from Tracks List

We will implement a premium and seamless experience to allow users to rename tracks or recorded points directly from the **Tracks List** screen.

## User Experience Design
1. **Interactive UI Elements**:
   - An elegant "Edit/Rename" pencil icon button (`btnRename`) will be added to the row item of each track/point in `item_track.xml`, positioned harmoniously alongside the Map, Share, and Delete buttons.
   - The icon will use Android's standard `@android:drawable/ic_menu_edit` (pencil) with the application's premium accent color `#0F3460`.
2. **Interactive Dialog**:
   - Clicking the Rename button will display a custom styled dialog.
   - The dialog will dynamically adapt its title (e.g., "Rename Track" vs "Rename Point") depending on whether the item is a single point or a full track.
   - The input `EditText` in the dialog will be pre-populated with the clean existing description of the track/point (stripping standard structural prefix/timestamps like `gps_`, `location_logs_`, `Recorded Point`, etc., so the user only edits the human-readable part).
   - The input will have high-fidelity styling (using the app's standard `bg_edit_text.xml` border and background selector, premium padding, and matching typography).
3. **Data Integrity & Robust Renaming**:
   - Each track item can represent multiple files (e.g., `.gpx`, `.kml`, `.json`).
   - The renaming action will safely rename **all associated format files** of the selected item in the background.
   - It checks for existing destination filenames to avoid silent overwrites or conflicts.
   - It maintains the correct filename structure (`gps_[TIMESTAMP]_[description]`), preserving the original timestamp and prefix so sorting and list parsing remain intact and flawless.
   - Upon successful renaming, a Toast confirmation is shown and the list refreshes immediately.

- Target file: `app/src/main/java/com/example/gpslocationlogger/TracksListActivity.java`
