# Implementation Plan for Parsing Multiple Map Formats

**User Prompt:**
> show on map from GPX, KML or JSON file.  Check for files in that order

1. **Investigate the Problem**:
    * Currently, `MapActivity` only parses `.json` files to show the map.
    * The user wants to support reading from GPX or KML if available, specifically checking in the order: `.gpx`, then `.kml`, then `.json`.
    * This allows users to disable JSON export in settings but still use the map viewer.
    * We can parse the GPX and KML files using simple Regex, since they are generated locally in a fixed format by `MainActivity.java`.

2. **Implementation Steps**:
    * **`MapActivity.java`**:
        * In `loadTrackData()`, check for `file.gpx`, then `file.kml`, then `file.json`.
        * Read the file contents into a String.
        * For GPX: Use regex `<trkpt lat="([^"]+)" lon="([^"]+)">` to extract `lat` (group 1) and `lon` (group 2).
        * For KML: Extract coordinates from `<coordinates>...</coordinates>` block or simply search for `\s+([\d\.-]+),([\d\.-]+),0` which extracts `lon` (group 1) and `lat` (group 2).
        * For JSON: Keep the existing JSON array parsing.
        * Extract points, build the bounds, and draw the polyline exactly as before.
