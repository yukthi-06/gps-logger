# Implementation Plan - Coordinate Precision Discrepancy Fix

> When saving to file, The latitude and longitude values in GPX and KML are same, but the JSON values are different .

## The Problem
When the application saves GPS track logs to files, it creates three formats (GPX, KML, JSON) representing the same track data. However, there are two distinct discrepancies between the saved coordinates:
1. **Precision Mismatch**: GPX and KML formatters truncate/round latitude and longitude to 6 decimal places (`%.6f`), whereas JSON stores the full, raw double-precision coordinates directly from the `Location` object (usually 14-15 decimal places).
2. **Locale Dependency Risk**: GPX and KML generators use `String.format("%.6f", value)` without specifying `Locale.US`. On devices with European locales (where the decimal separator is a comma `,` instead of a dot `.`), this outputs coordinates like `12,345678` in GPX and KML files. Since the GPX and KML standards strictly require a dot `.` as the decimal separator, these files become corrupted. Furthermore, the regex parsers in `MapActivity` and `SharePointHelper` (which look for dots or specific number formats) fail to extract coordinates from these malformed files, throwing `NumberFormatException`.

## The Solution
To make the coordinate values completely identical and correct across GPX, KML, and JSON:
1. Avoid truncating coordinates to 6 decimal places in GPX and KML. Geographic coordinates should be written with full precision to match the JSON output and maintain maximum logging accuracy.
2. Use a locale-independent representation (such as `Double.toString(value)` or `String.format(Locale.US, ...)`). `Double.toString(value)` is clean, fast, handles full precision, and always outputs with a dot `.` as the decimal separator.

### Proposed Changes

#### 1. `MainActivity.java`
Update `generateGpx` and `generateKml` to output coordinates using `Double.toString()` instead of the locale-sensitive, truncated `String.format("%.6f", ...)`:

- **In `generateGpx`**:
  Replace:
  ```java
  sb.append(String.format("      <trkpt lat=\"%.6f\" lon=\"%.6f\">\n", lat, lon));
  ```
  With:
  ```java
  sb.append("      <trkpt lat=\"").append(Double.toString(lat)).append("\" lon=\"").append(Double.toString(lon)).append("\">\n");
  ```

- **In `generateKml`**:
  Replace:
  ```java
  sb.append(String.format("          %.6f,%.6f,0\n", lon, lat));
  ```
  With:
  ```java
  sb.append("          ").append(Double.toString(lon)).append(",").append(Double.toString(lat)).append(",0\n");
  ```

---

By applying this change, the exact same raw `double` coordinate values will be outputted to GPX, KML, and JSON files without any locale-specific issues.
