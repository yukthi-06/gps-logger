# Implementation Plan - Immediate Settings Availability & Real-Time Sync

We will ensure that any settings modifications—whether made in the Options UI or modified in the external `/sdcard/Vypeensoft/GPS_Location_Logger/settings/settings.json` file directly—are immediately available to all active components of the application (including the active background `LocationService`).

## Solution Design

### 1. Dynamic GPS Frequency Updates in Background
Currently, the background `LocationService` requests GPS coordinates via `FusedLocationProviderClient` with the logging interval specified when the tracking started. If the user changes the frequency in the Options UI, the running background service continues tracking at the old rate.
- **Solution**: Implement `SharedPreferences.OnSharedPreferenceChangeListener` in `LocationService.java`.
- When the `logging_interval_ms` preference key changes, the listener will instantly remove current GPS updates and re-register them with the new interval by calling `startLocationUpdates()`. This applies the changes in real time without stopping the tracking session.

### 2. Immediate Settings Re-sync on App Resume
If the `/sdcard/Vypeensoft/GPS_Location_Logger/settings/settings.json` file is modified directly (e.g., via a text editor or PC transfer), the app should immediately import the changes.
- **Solution**: Trigger `SettingsHelper.loadSettingsFromJson(this, prefs)` inside `MainActivity.onResume()` and `SettingsActivity.onResume()`.
- This ensures that as soon as the user returns to the app, the external JSON settings are loaded. Any updated frequency will propagate to `SharedPreferences`, which in turn triggers the `LocationService` listener to update the background tracking rate instantly.

## Proposed Changes

### 1. Update `LocationService.java`
- Implement `SharedPreferences.OnSharedPreferenceChangeListener`.
- In `onCreate()`, register the listener.
- In `onDestroy()`, unregister the listener.
- In `onSharedPreferenceChanged(sharedPreferences, key)`:
  - If `key.equals(SettingsActivity.KEY_INTERVAL_MS)` and the service is actively tracking (not paused), remove current location callback and re-run `startLocationUpdates()`.

### 2. Update `MainActivity.java`
- Implement/override `onResume()`.
- Inside `onResume()`, fetch `SharedPreferences` and run `SettingsHelper.loadSettingsFromJson(this, prefs)`.

### 3. Update `SettingsActivity.java`
- Implement/override `onResume()`.
- Inside `onResume()`, run `SettingsHelper.loadSettingsFromJson(this, prefs)` to load any external edits, and reload settings UI fields if changes occurred (dropdown selections, format checkboxes, map style URLs).
