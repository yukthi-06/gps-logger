# Implementation Plan - Exclude & Remove x86 Libraries

We will configure the Gradle build script to completely exclude and purge all C++ native binaries for `x86` and `x86_64` CPU architectures. This will prevent compiling any x86 code, shrinking all generated output packages (including the universal `app-debug.apk`) to their absolute minimum sizes by only bundling ARM libraries.

## Solution Design

1. **Global NDK ABI Filtering**:
   - Inside `defaultConfig` in `app/build.gradle`, we will add the `ndk.abiFilters` block, defining only the ARM-based CPU architectures: `"armeabi-v7a"` and `"arm64-v8a"`.
   - This prevents packaging any `x86` or `x86_64` native `.so` library engines during compile/link stages, completely stripping them from the universal APK (`app-debug.apk`).
   - Consequently, the universal `app-debug.apk` will plunge in size from `58MB` down to approximately **`27MB`**!

2. **Split ABI Exclusions**:
   - Under `splits.abi.include`, we will remove `"x86"` and `"x86_64"` so that splits are generated only for ARM devices.

## Proposed Changes

### 1. Update `app/build.gradle`
- Add `ndk` configuration inside `defaultConfig`:
```groovy
        ndk {
            abiFilters "armeabi-v7a", "arm64-v8a"
        }
```
- Update `splits.abi.include` to exclude x86 architectures:
```groovy
            include "armeabi-v7a", "arm64-v8a"
```
