# Implementation Plan - Track List Visual Indicators and Location Mapping

This document details the architecture, design choices, and implementation details for enabling users to visually differentiate between recorded individual GPS points and continuous track logs, and how the map automatically renders single points with a marker.

## 1. Visual Indicators & Differentiation on Track List

### Model Level
- Added `isRecordedPoint()` helper method inside `TrackItem.java`.
- Detection logic: Checks if `baseName` contains the string substring `_Recorded_Point`. This ensures a highly robust, zero-overhead identification of single-point logs on the fly, since single points are named `location_logs_[timestamp]_Recorded_Point_[notes].[ext]`.

### Layout Level (`item_track.xml`)
- Assigned the unique ID `@+id/ivTrackIcon` to the left-aligned `ImageView` inside `item_track.xml` (which previously lacked an ID).

### Adapter Level (`TrackAdapter.java`)
- Imported `android.widget.ImageView`.
- Declared and bound `ivTrackIcon` inside the ViewHolder.
- Inside `onBindViewHolder()`:
  - If `trackItem.isRecordedPoint()` is `true`:
    - The icon resource is programmatically set to `android.R.drawable.ic_menu_mylocation` (a modern, circular crosshair GPS target).
    - The title text is prepended with a location pin emoji `📍 ` (e.g. `📍 Recorded Point My Notes`).
  - If `trackItem.isRecordedPoint()` is `false`:
    - The icon resource is programmatically set to `android.R.drawable.ic_menu_mapmode` (representing a path / track layout).
    - The title text is prepended with a checkered finish flag emoji `🏁 ` (e.g. `🏁 Continuous Track Logs`).

### Deletion Polish (`TracksListActivity.java`)
- Enhanced the `onDeleteClick()` confirmation dialog text to dynamically use `"Delete Recorded Point"` or `"Delete Track"` instead of a hardcoded `"Delete Track"`.

## 2. Dynamic Single Point Marker Rendering on Map

### Map Activity Click Handling (`MapActivity.java` integration)
- When a user clicks on an item in the RecyclerView (or clicks its map button), `TracksListActivity` starts `MapActivity` with the track file basename.
- Inside `MapActivity.java`:
  - The GPX regex parser is updated to accurately read both waypoint `<wpt>` and trackpoint `<trkpt>` tags.
  - If the points array contains exactly `1` point (`points.size() == 1`), `MapActivity` bypasses standard line drawing (`LineManager` / `LineOptions`) and instead invokes the MapLibre annotations circle plugin:
    - Uses `CircleManager` and `CircleOptions` to render a solid, beautiful red circle marker (`#D32F2F`) at that exact coordinate.
    - Standardized circle aesthetics: `10.0f` radius, styled with a high-contrast `2.0f` thick white (`#FFFFFF`) stroke border to look premium and highly visible.
  - The camera bounding box automatically centers and zooms directly over the point.
