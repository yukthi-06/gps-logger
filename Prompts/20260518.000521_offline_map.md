# Implementation Plan for Offline Map Tiles

**User Prompt:**
> download the relevant map tiles for offline use

1. **Investigate the Problem**:
    * MapLibre SDK supports offline maps via the `OfflineManager`.
    * To download tiles, we must define an `OfflineTilePyramidRegionDefinition` which requires a Style URI, bounds, and zoom levels.
    * Currently, we set the map style inline using a JSON string. We need to move this to an asset file `osm_style.json` so we can reference it via `"asset://osm_style.json"` for both the MapView and the `OfflineManager`.
    * We should add a "Download Offline" action to the `MapActivity` so users can download the bounding box of their route.

2. **Implementation Steps**:
    * **`app/src/main/assets/osm_style.json`**: Create a file containing the OpenStreetMap raster style JSON.
    * **`app/src/main/res/menu/menu_map.xml`**: Create a new menu for `MapActivity` with a "Download Offline" item.
    * **`MapActivity.java`**:
        * Update `onMapReady()` to use `Style.Builder().fromUri("asset://osm_style.json")`.
        * Save the `LatLngBounds` to a class variable.
        * Override `onCreateOptionsMenu` and `onOptionsItemSelected` to inflate the menu and handle the download button.
        * In the download handler, construct `OfflineTilePyramidRegionDefinition` using the bounds and zoom levels (e.g., 10 to 18).
        * Use `OfflineManager` to create the region and observe download progress via `OfflineRegionObserver`.
        * Show Toast messages to indicate download start and completion.
