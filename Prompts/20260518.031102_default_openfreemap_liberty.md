# Implementation Plan for Default Map Style Migration (OpenFreeMap Liberty)

**User Prompt:**
> In settings, set the Map tiles url to 'https://tiles.openfreemap.org/styles/liberty' as the default value

1. **Investigate the Problem**:
    * The user wants to change the default Map Style URL in settings to OpenFreeMap Liberty (`https://tiles.openfreemap.org/styles/liberty`).
    * If we only change the default constant, existing users who already ran the app will still have their local preference saved to the old Carto Voyager URL (`https://basemaps.cartocdn.com/gl/voyager-gl-style/style.json`).
    * We should implement a **dynamic preference migration** that updates the saved preference to the new default if it matches the old default value. This ensures a seamless update for all existing users!

2. **Implementation Steps**:
    * **`SettingsActivity.java`**:
        * Change `DEFAULT_MAP_STYLE_URL` to `"https://tiles.openfreemap.org/styles/liberty"`.
        * Define `OLD_DEFAULT_MAP_STYLE_URL` as `"https://basemaps.cartocdn.com/gl/voyager-gl-style/style.json"`.
        * Inside `onCreate()`, if the saved style matches `OLD_DEFAULT_MAP_STYLE_URL`, automatically migrate the saved preference value to `DEFAULT_MAP_STYLE_URL` and save it.
    * **`MapActivity.java`**:
        * Inside `onCreate()`, do a similar check: if the saved style in SharedPreferences matches the old default, migrate it dynamically to the new OpenFreeMap Liberty default value, saving it in the preference and updating `styleUrl`.
