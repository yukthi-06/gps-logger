# Implementation Plan for Second Line Action Buttons in Map Screen

**User Prompt:**
> The track name text along with Share icon, Delete icon, Download icon is too cramped.
> Move the buttons to a second line and keep the top line just for the name

1. **Investigate the Problem**:
    * Currently, the Share, Delete, and Download icons are placed in the Toolbar (Action Bar) next to the dynamic track name.
    * When the track name is long, the Toolbar space is extremely cramped, squishing or cutting off the text.
    * To resolve this, we will move the action buttons out of the Toolbar into a dedicated, premium **second line** action bar located directly below the Toolbar.
    * This second line action bar will feature three beautifully spaced outline-styled buttons:
      * **Share** (Blue border, share icon, blue text)
      * **Offline** (Blue border, download icon, blue text)
      * **Delete** (Red border, delete icon, red text)
    * This allows the top Toolbar to be dedicated solely to the route name, offering perfect readability and look-and-feel.

2. **Implementation Steps**:
    * **`app/src/main/res/drawable/bg_action_button.xml`**: Create a custom ripple-backed outline background shape for primary actions (Share, Offline) using `#0F3460`.
    * **`app/src/main/res/drawable/bg_action_button_delete.xml`**: Create a custom ripple-backed outline background shape for the delete action using `#D32F2F`.
    * **`app/src/main/res/layout/activity_map.xml`**: Add the horizontal actions layout `llMapActions` and a thin `#E2E8F0` divider line right below the Toolbar and above the MapView FrameLayout.
    * **`MapActivity.java`**:
      * Remove the menu inflation methods `onCreateOptionsMenu()` and `onOptionsItemSelected()`.
      * Inside `onCreate()`, bind click listeners for the three buttons: `btnShareMap`, `btnDeleteMap`, and `btnDownloadMap`.
