# Implementation Plan for Configurable Map Style URL

**User Prompt:**
> make the Map Style URL as configurable from the settings page

1. **Investigate the Problem**:
    * Currently, `MapActivity.java` defines the `STYLE_URL` as a hardcoded private static final String constant.
    * To make it configurable, we need to:
      1. Declare a SharedPreferences key for the Map Style URL in `SettingsActivity.java` (shared with `MapActivity.java`).
      2. Add a Map Settings section in `activity_settings.xml` with a `CardView` and a text input field (`TextInputLayout`/`EditText`) for the URL.
      3. Read and populate this input field from SharedPreferences inside `SettingsActivity.java`, and save the value dynamically when modified.
      4. Read the map style URL from SharedPreferences inside `MapActivity.java`, falling back to the CARTO Voyager GL style if no custom URL is specified.

2. **Implementation Steps**:
    * **`res/values/strings.xml`**: Add strings for the new "MAP SETTINGS" section, "Map Style URL" label, and description.
    * **`res/layout/activity_settings.xml`**:
        * Add a new section below "Export Formats" with the header "MAP SETTINGS".
        * Insert a `CardView` containing a `TextInputLayout` and an `EditText` (`etMapStyleUrl`) styled consistently with the rest of the options page.
    * **`SettingsActivity.java`**:
        * Define `public static final String KEY_MAP_STYLE_URL = "map_style_url";`.
        * Define the default fallback: `public static final String DEFAULT_MAP_STYLE_URL = "https://basemaps.cartocdn.com/gl/voyager-gl-style/style.json";`.
        * Bind the `EditText`, load the saved preference, and attach a `TextWatcher` to dynamically write value updates back to `SharedPreferences`.
    * **`MapActivity.java`**:
        * Retrieve the `styleUrl` from SharedPreferences when the map loads, falling back to the configured default value.
        * Use this dynamic `styleUrl` inside both the MapView style setup and the offline downloader definition.
