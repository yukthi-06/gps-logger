# Implementation Plan for Multi-Format Share and ZIP Package

**User Prompt:**
> On 'Share" of a track, show checkboxes for "gpx, kml, json". If more than one option is selected, zip those files and proceed for sharing. If only one item is selected, then share that file as it is....

1. **Investigate the Problem**:
    * Currently, track sharing only picks the "best" format (prioritizing KML, then GPX, then JSON) and shares that single file.
    * The user wants to choose which formats to share using checkboxes if multiple files exist.
    * If only one format is checked, share that file directly.
    * If more than one format is checked, pack them into a standard `.zip` archive on the fly and share the ZIP file!
    * To ensure a flawless, uncluttered storage folder, we will create temporary ZIP files inside the application's secure cache directory (`getExternalCacheDir()`), which is automatically cleaned by Android as needed.
    * We should implement this consistently in both:
      * **`TracksListActivity.java`** (when sharing from the recycler view track list item).
      * **`MapActivity.java`** (when sharing from the second-line horizontal Action Bar).

2. **Implementation Steps**:
    * **`TracksListActivity.java`**:
      * Add imports for ZIP and IO buffer streams: `java.io.BufferedInputStream`, `java.io.BufferedOutputStream`, `java.io.FileOutputStream`, `java.io.IOException`, `java.util.zip.ZipEntry`, `java.util.zip.ZipOutputStream`.
      * Update `onShareClick()` to discover available files, show a custom checkbox `AlertDialog`, and compile the zipped package or single-file export on-demand.
      * Add helper methods `shareSingleFile()` and `zipFiles()`.
    * **`MapActivity.java`**:
      * Import same IO and ZIP libraries.
      * Replace `shareTrack()` to mirror the exact same multi-choice checkboxes, zipping, and sharing flow.
