# Implementation Plan for Logging Frequency Dropdown Spinner

**User Prompt:**
> Logging frequency in settings - make it a proper dropdown.

1. **Investigate the Requirements**:
    * Currently, the settings page features 8 different logging frequencies (1s, 2s, 3s, 5s, 10s, 30s, 1m, 5m) presented inside a long `RadioGroup`.
    * This occupies huge vertical space, creating an untidy UI.
    * Replacing it with a beautiful, compact **Material Dropdown Spinner (`AppCompatSpinner`)** saves significant space and provides a polished interface.

2. **Implementation Steps**:
    * **`activity_settings.xml`**: Replace the `RadioGroup` and its 8 `RadioButton` children with a stylish `androidx.appcompat.widget.AppCompatSpinner` using `#0F3460` (Primary Theme Navy) as its `backgroundTint` dropdown indicator.
    * **`SettingsActivity.java`**:
      * Change the `rgFrequency` field to `Spinner spFrequency`.
      * Clean up the obsolete `RADIO_IDS` array, `selectRadioForInterval` method, and `intervalForRadioId` method.
      * Set up an `ArrayAdapter` to populate the `Spinner` with polished labels (incorporating user-friendly subtexts like "Highest frequency" and "Battery saver").
      * Bind the spinner's `OnItemSelectedListener` to store selected frequencies inside SharedPreferences (`KEY_INTERVAL_MS`) and update the live preview label below.
