# Implementation Plan for Requesting File Permissions on Tracks List

**User Prompt:**
> It says "Folder is empty".  File read permission is not asked at start of app

1. **Investigate the Problem**:
    * The user noted that the read permission is not asked when the app starts or when they go to the Tracks List.
    * In `MainActivity.java`, the permission is requested only when the user clicks "Start Tracking".
    * If the user hasn't clicked "Start Tracking" yet, `TracksListActivity` tries to read the folder without permission, resulting in `files` returning empty or null (depending on device).
    
2. **Implementation Steps**:
    * Update `TracksListActivity.java` to explicitly check for `MANAGE_EXTERNAL_STORAGE` (API 30+) or `READ_EXTERNAL_STORAGE` (API < 30) before attempting to read the directory.
    * If permissions are missing, request them directly in `TracksListActivity`.
    * If the user grants permission, reload the tracks.
    * Ensure we use the robust path `Environment.getExternalStorageDirectory() + "/Vypeensoft/GPS_Location_Logger"` rather than just hardcoding `/sdcard/` to be safe across different OEMs, although we will keep the hardcoded message if it helps. I will switch back to `Environment.getExternalStorageDirectory()` for safety and append the rest of the path.
