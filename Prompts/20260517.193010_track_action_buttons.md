# Implementation Plan for Track Item Action Buttons

**User Prompt:**
> On each track item, show small button icons for "Show on map", "Share", "Delete".

1. **Investigate the Problem**:
    * Currently, clicking the entire track item automatically shares it. The user wants explicit buttons for "Map", "Share", and "Delete".
    * The track card needs UI updates to house these buttons (e.g., a horizontal `LinearLayout` of `ImageButton`s).
    * `TrackAdapter` needs to handle clicks on these specific buttons instead of the whole item.
    * `TracksListActivity` needs logic to:
        * **Map**: Open the best file (preferably KML or GPX) using an `ACTION_VIEW` intent with appropriate MIME types (`application/vnd.google-earth.kml+xml` for KML, `*/*` or `application/gpx+xml` for GPX).
        * **Share**: Move the existing share logic here.
        * **Delete**: Delete all associated files (`.gpx`, `.kml`, `.json`) with the given `baseName`, then refresh the list.

2. **Implementation Steps**:
    * **`item_track.xml`**:
        * Add a `LinearLayout` inside the main `LinearLayout` of the `CardView`.
        * Add three `ImageButton`s using standard Android drawable icons:
            * Map: `@android:drawable/ic_dialog_map`
            * Share: `@android:drawable/ic_menu_share`
            * Delete: `@android:drawable/ic_menu_delete`
    * **`TrackAdapter.java`**:
        * Define a new interface `TrackActionListener` with methods `onMap(TrackItem item)`, `onShare(TrackItem item)`, `onDelete(TrackItem item)`.
        * Update the constructor and `ViewHolder` to bind the new buttons and listeners.
    * **`TracksListActivity.java`**:
        * Implement the `TrackActionListener` interface.
        * Extract share logic from `onTrackClicked` to `onShare`.
        * Add `onMap` logic to dispatch a View intent for KML/GPX.
        * Add `onDelete` logic to show a confirmation dialog, delete files matching `baseName`, and call `loadTracks()`.
