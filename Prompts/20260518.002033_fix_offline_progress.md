# Implementation Plan for Fixing Offline Map Progress

**User Prompt:**
> the screen is stuck at 'calculating download size...'

1. **Investigate the Problem**:
    * MapLibre's `OfflineRegionStatus` exposes a method `isRequiredResourceCountPrecise()`.
    * The current logic waits for this to be `true` before displaying actual numerical progress.
    * However, MapLibre often keeps this `false` during the majority of the download because it discovers tile dependencies dynamically. As a result, the UI appears stuck in the "Calculating" state even though tiles are actively downloading in the background.

2. **Implementation Steps**:
    * **`MapActivity.java`**:
        * Inside `onStatusChanged(OfflineRegionStatus status)`, remove the strict `if (status.isRequiredResourceCountPrecise())` check.
        * Instead, check `if (status.getRequiredResourceCount() > 0)`.
        * This allows the app to report the estimated (but constantly updating and accurate) progress.
        * Wrap the UI updates inside `runOnUiThread()` to absolutely ensure they are executed on the main thread and avoid any potential silent `ViewRootImpl` exceptions that could freeze the UI updates.
