# Implementation Plan for Formatting Tracks List Timestamps

**User Prompt:**
> Remove the millisecond from the display...
> If the Timestamp is the main text, then subtitle need not display the timestamp again.

1. **Investigate the Problem**:
    * Currently, the parsed timestamp can contain milliseconds and time zone offsets like `.123+05-30` or `+05-30`. 
    * We should truncate everything after the first `.` or `+` to simplify the string.
    * Also, `T` and `-` in the time portion can be formatted nicer (e.g., `2026-05-17 18:15:09` instead of `2026-05-17T18-15-09`).
    * When there's no trip info, `displayTimestamp` shouldn't show the same info as `displayName` (subtitle hidden).

2. **Implementation Steps**:
    * Create a helper method in `TracksListActivity.java` to format the timestamp:
        * Drop any characters from `.` or `+` onwards.
        * Replace `T` with a space ` `.
        * Replace `-` with `:` for the time portion. The date portion has `-`, so we should only replace `-` after the space.
    * Update `TracksListActivity.java` to set `item.displayTimestamp = null` if there is no tracking info.
    * Update `TrackAdapter.java` to hide `tvTrackTimestamp` (`View.GONE`) if `trackItem.displayTimestamp` is null or empty, and show it (`View.VISIBLE`) otherwise.
