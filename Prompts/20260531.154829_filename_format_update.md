# Implementation Plan - Filename Format and Prefix Update

> for the output filename, remove the millisecond and timezone .
> Also remove "location_logs" prefix and replace with "gps"

## Proposed Changes

### 1. `MainActivity.java`
Update the filename generation logic in both `saveSinglePointToFile` and `saveLocationDataToFile`:

- **Change Timestamp Formatter**:
  Replace:
  ```java
  String fileTimestamp = ZonedDateTime.now()
          .format(DateTimeFormatter.ISO_OFFSET_DATE_TIME)
          .replace(":", "-");
  ```
  With a custom pattern that formats up to seconds (without milliseconds or timezone):
  ```java
  String fileTimestamp = ZonedDateTime.now()
          .format(DateTimeFormatter.ofPattern("yyyy-MM-dd'T'HH-mm-ss"));
  ```

- **Update Prefix**:
  Replace the prefix `"location_logs_"` with `"gps_"`.
  - In `saveSinglePointToFile`:
    ```java
    baseName = "gps_" + fileTimestamp + "_Recorded_Point_" + sanitizedInfo;
    // ... or without sanitizedInfo:
    baseName = "gps_" + fileTimestamp + "_Recorded_Point";
    ```
  - In `saveLocationDataToFile`:
    ```java
    baseName = "gps_" + fileTimestamp + "_" + sanitizedInfo;
    // ... or without sanitizedInfo:
    baseName = "gps_" + fileTimestamp;
    ```

### 2. `TracksListActivity.java`
Ensure the file list displays both new tracks (prefixed with `"gps_"`) and existing tracks (prefixed with `"location_logs_"`) correctly:

- **Update `parseTrackItemDetails`**:
  Instead of only looking for `"location_logs_"`, check if the filename starts with `"gps_"` first:
  ```java
  private void parseTrackItemDetails(TrackItem item) {
      String name = item.baseName;
      String prefix = "location_logs_";
      if (name.startsWith("gps_")) {
          prefix = "gps_";
      }
      if (name.startsWith(prefix)) {
          // ... rest of the parsing is dynamic and works perfectly
      }
  }
  ```

This ensures full backwards compatibility with older track files while seamlessly adopting the clean new name format.
