# Implementation Plan for Fixing Stuck Offline Map Downloads

**User Prompt:**
> No change... the screen is still stuck at 'calculating download size...'

1. **Investigate the Problem**:
    * The download process is getting stuck at "calculating download size...".
    * In MapLibre / Mapbox SDKs, setting the download state `setDownloadState(OfflineRegion.STATE_ACTIVE)` before setting the observer `setObserver(...)` is a classic race condition. The engine begins starting the download and queries tile counts before the observer is attached. As a result, the progress callbacks never fire, leaving the UI permanently in the initial "Calculating" state.
    * We must attach the `OfflineRegionObserver` *first*, and then activate the download state.
    * We will also add explicit logging to track what values are being passed and if there are any exceptions.

2. **Implementation Steps**:
    * **`MapActivity.java`**:
        * Rearrange the `OfflineRegion` setup in `downloadOfflineMap()`:
            1. First, call `offlineRegion.setObserver(...)`.
            2. Second, call `offlineRegion.setDownloadState(OfflineRegion.STATE_ACTIVE)`.
        * Add `Log.d` logs inside `onStatusChanged` to print `completedResourceCount` and `requiredResourceCount` so we can inspect it in logcat if needed.
