# Implementation Plan: Fix Crash on End Tracking & Update File I/O Logic

## Objective
Fix the application crash that occurs when the user clicks "End tracking", and ensure that the final GPX and other log files are correctly saved to `/sdcard/Vypeensoft/GPS_Location_Logger`.

## Cause of Crash
When the file saving path was updated to `Vypeensoft/GPS_Location_Logger` in the `MediaStore.Downloads.RELATIVE_PATH`, it caused an `IllegalArgumentException` on Android 10+ devices because `MediaStore.Downloads` only allows paths that are subdirectories of the standard `Download/` or `Downloads/` directory.

## Changes Made
1. **`AndroidManifest.xml`**:
   - Added `WRITE_EXTERNAL_STORAGE` permission for Android versions below 11 (API < 30).
   - Added `MANAGE_EXTERNAL_STORAGE` permission for Android 11+ (API >= 30) to allow writing to arbitrary directories at the root of the SD card.
   - Declared a `FileProvider` so that generated log files can be securely shared with other apps using `content://` URIs instead of the insecure `file://` URIs.

2. **`res/xml/provider_paths.xml`**:
   - Created this configuration file to declare the external paths that the `FileProvider` is allowed to share (`<external-path name="external_files" path="." />`).

3. **`MainActivity.java`**:
   - Removed `saveViaMediaStore()` which was constrained by scoped storage relative paths.
   - Implemented a new `saveToFile()` method that uses standard `java.io.File` API to write directly to `/sdcard/Vypeensoft/GPS_Location_Logger`.
   - Updated `saveToFile()` to return a secure `content://` URI using `FileProvider.getUriForFile()`, enabling seamless operation of the "Share" intent.
   - Updated the `checkAndRequestPermission()` method to prompt the user to grant the `All files access` (MANAGE_EXTERNAL_STORAGE) permission on Android 11+ devices via the system settings screen if it hasn't been granted yet. Also included standard runtime permission requests for `WRITE_EXTERNAL_STORAGE` on devices running Android 10 or below.
