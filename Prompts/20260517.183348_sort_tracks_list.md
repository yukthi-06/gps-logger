# Implementation Plan for Sorting Tracks List

**User Prompt:**
> Tracks list has to be sorted by timestamp in descending order.....(newest on one top)

1. **Investigate the Problem**:
    * The current sorting mechanism relies on `file.lastModified()`.
    * If files are copied, modified, or restored, the OS's `lastModified` date can change and may not accurately represent the chronological recording order.
    * The track baseName naturally contains the ISO timestamp right after `location_logs_` (e.g., `location_logs_2026-05-17T18-15-09`).
    * The string representation of this timestamp allows for natural lexicographical sorting which perfectly aligns with chronological order.

2. **Implementation Steps**:
    * Update `TracksListActivity.java` to modify the `Collections.sort` logic.
    * Instead of sorting by `lastModified`, sort by the `baseName` or a parsed string representation in descending order.
    * Using `baseName` directly: `t2.baseName.compareTo(t1.baseName)` will perfectly sort it descending, since the `location_logs_` prefix is static, and the next characters are the ISO year, month, day, hour, minute, second.
    * Apply this change and re-notify the adapter.
