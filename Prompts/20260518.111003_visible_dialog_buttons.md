# Implementation Plan for Visible Dialog Buttons (High Contrast Styling)

**User Prompt:**
> DELETE alterbox buttons, SHARE alertbox buttons are dark in color and no contrast. change to more visible colors

1. **Investigate the Problem**:
    * Android's default `AlertDialog` buttons take their accent styling color from the parent theme. On some configurations, this results in dark-grey or black button text with zero contrast, making buttons nearly invisible.
    * To resolve this, we will programmatically retrieve the created dialog buttons (`BUTTON_POSITIVE` and `BUTTON_NEGATIVE`) immediately after calling `show()`.
    * We will color them using beautiful, rich, high-contrast HSL-aligned color tokens:
      * **Positive/Neutral Action (Share)**: `#0F3460` (Primary Theme Blue).
      * **Destructive Action (Delete/Clear)**: `#D32F2F` (Vivid Red).
      * **Negative/Dismiss Action (Cancel)**: `#888EAB` (Neutral Slate Grey).
    * This guarantees 100% visible, high-contrast, premium styled dialog buttons across all Android versions.

2. **Implementation Steps**:
    * **`SettingsActivity.java`**: Style the "Clear Offline Cache" dialog buttons (Clear -> `#D32F2F`, Cancel -> `#888EAB`).
    * **`TracksListActivity.java`**:
      * Style the "Share track formats" dialog buttons (Share -> `#0F3460`, Cancel -> `#888EAB`).
      * Style the "Delete track" dialog buttons (Delete -> `#D32F2F`, Cancel -> `#888EAB`).
    * **`MapActivity.java`**:
      * Style the "Share track formats" dialog buttons (Share -> `#0F3460`, Cancel -> `#888EAB`).
      * Style the "Delete track" dialog buttons (Delete -> `#D32F2F`, Cancel -> `#888EAB`).
