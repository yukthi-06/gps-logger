# Implementation Plan for Map View

**User Prompt:**
> on clikcing on 'map' button, it should open a new screen and load a map(from libre map library) and show the route on that map.  The map must be zoomable 

1. **Investigate the Problem**:
    * Currently, the "Map" button in `TracksListActivity` tries to open an external app via an `ACTION_VIEW` intent.
    * The user wants to see the map *inside* the app itself using a free/libre map library.
    * We will use `osmdroid`, the standard open-source OpenStreetMap library for Android.
    * We need to add the `osmdroid` dependency.
    * We need to create a new `MapActivity` that receives the track name, loads its location data, and draws a `Polyline` on a `MapView`.

2. **Implementation Steps**:
    * **`app/build.gradle`**: Add `implementation 'org.osmdroid:osmdroid-android:6.1.18'`.
    * **`AndroidManifest.xml`**: Register `MapActivity` as a child of `TracksListActivity`. Ensure internet permission and network state permissions exist (usually they do).
    * **`activity_map.xml`**: Add `org.osmdroid.views.MapView` with a `Toolbar` at the top.
    * **`MapActivity.java`**:
        * Initialize `osmdroid` configuration.
        * Read the track file. Since the app can save in `.json`, `.gpx`, or `.kml`, we will look for `.json` first for easy native parsing. If missing, we'll parse `.gpx` or `.kml` using simple regex or string splitting, since the app generates these in a predictable format.
        * Create `GeoPoint`s for each coordinate.
        * Draw a `Polyline` connecting the `GeoPoint`s on the map.
        * Zoom the map to the bounding box of the route.
    * **`TracksListActivity.java`**: Update `onMapClick()` to launch `MapActivity` instead of an external intent. Pass the `baseName` via `Intent.putExtra()`.
