# Implementation Plan: Move Project Data Folder

## Objective
Move the project settings and data folder base from `/sdcard/Downloads/GPSLocationLogger` to `/sdcard/Vypeensoft/GPS_Location_Logger`.

## Changes Made
1. **`MainActivity.java`**:
   - Updated the `GPS_FOLDER` constant from `"GPSLocationLogger"` to `"Vypeensoft/GPS_Location_Logger"`.
   - Removed `Environment.DIRECTORY_DOWNLOADS + File.separator` from the `RELATIVE_PATH` logic in `saveViaMediaStore()` so that it now natively tries to save directly to `Vypeensoft/GPS_Location_Logger` instead of appending it inside the `Download` directory.
   - Updated JavaDoc comments and inline documentation to accurately reflect the new path `/sdcard/Vypeensoft/GPS_Location_Logger/`.

2. **`strings.xml`**:
   - Updated the string resource `label_save_path` to reflect the new directory path `Vypeensoft/GPS_Location_Logger/`.

## Potential Impacts (API 29+ Scoped Storage)
Note: Due to Android 10+ (API 29) scoped storage restrictions, modifying `MediaStore.Downloads.RELATIVE_PATH` to point outside of the standard `Download/` or `Documents/` directories may throw an `IllegalArgumentException` without `MANAGE_EXTERNAL_STORAGE` permission on modern Android versions. If the application crashes when saving files, you may need to:
1. Ensure the app has the legacy external storage request enabled if targeting API 29.
2. Direct the path via `MediaStore.Files` or update permissions to include `MANAGE_EXTERNAL_STORAGE` on API 30+.
