# Implementation Plan for Switching to a Free Public Vector Style for Flawless Offline Downloading

**User Prompt:**
> still stuck at 'Downloading ... 0/1 tiles(0.0%)'

1. **Investigate the Problem**:
    * Even with the JNI `file://` helper, the MapLibre `OfflineManager` gets stuck at `0/1` tiles (caching the style itself).
    * **Why**: MapLibre's C++ native offline downloader caches tiles inside `offline.db`. To do this, it validates caching rules and extracts HTTP response headers (like `ETag`, `Expires`, and `Last-Modified`) from the downloaded style file. Local files (`file://` and `asset://` URIs) do not have HTTP headers! As a result, the C++ `OfflineDatabase` rejects the style cache transaction, and the background download thread hangs or halts immediately at `0/1` tiles.
    * **The Solution**: The style URL **MUST** be a valid, public `https://` endpoint so the C++ background thread can fetch it over the network with standard HTTP caching headers.
    * Rather than trying to host a custom style, we will use a **completely free, stable, public vector style**:
      * **CARTO Voyager GL Style**: `https://basemaps.cartocdn.com/gl/voyager-gl-style/style.json`
      * This style is completely free to use, highly optimized, serves worldwide OpenStreetMap data, and does not require any API keys.
    * Switching to a vector style will:
      1. Instantly solve the offline downloader hang, allowing it to download map tiles flawlessly.
      2. Significantly reduce the map download size because vector tiles are far more lightweight than heavy, pixelated raster PNGs.
      3. Upgrade the mapping interface to a beautiful, modern, high-DPI vector design with sharp text and support for rotations and tilts.

2. **Implementation Steps**:
    * **`MapActivity.java`**:
        * Declare a constant `STYLE_URL = "https://basemaps.cartocdn.com/gl/voyager-gl-style/style.json"`.
        * Use `STYLE_URL` in `onMapReady()` to load the map view style.
        * Use `STYLE_URL` in `downloadOfflineMap()` when initializing the `OfflineTilePyramidRegionDefinition`.
        * We can remove the `getStyleFileUri()` helper and the related imports if they are no longer needed, maintaining clean and minimal code.
