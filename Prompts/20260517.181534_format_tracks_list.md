# Implementation Plan for Formatting Tracks List Display Names

**User Prompt:**
> Dont show the full filename in the list item text... extract the tracking_info  portion of the file name.. (ie.. skip the 'location_log', skip the time stamp). 
> If there is no user provided tracking info data, then show the timestamp as list item text.
> Along with list item name, show timestamp as a small font below it

1. **Investigate the Problem**:
    * Current display name is just the full `baseName` (e.g., `location_logs_2026-05-17T18-15-09+05-30_My_Trip` or `location_logs_2026-05-17T18-15-09+05-30`).
    * The user wants to parse the filename to extract the `tripInfo` and `timestamp`.
    * Show `tripInfo` as the main title (or `timestamp` if `tripInfo` doesn't exist).
    * Show `timestamp` as a smaller subtitle below it.
    * Keep the file extensions info. We'll add another `TextView` to `item_track.xml` to hold the timestamp, or replace the layout appropriately.

2. **Implementation Steps**:
    * Update `TrackItem.java` to hold `displayName` and `displayTimestamp`.
    * Update `TracksListActivity.java` to parse `baseName` into these two new fields during the directory scanning loop.
        * If `baseName.startsWith("location_logs_")`, strip that.
        * Find the first `_` in the remaining string. The part before it is the `timestamp`. The part after it is the `tripInfo`.
        * Replace `_` with spaces in `tripInfo` for a cleaner display, since the original input had spaces replaced with `_`.
        * Handle edge cases where `_` doesn't exist (no trip info).
    * Update `item_track.xml` to add `tvTrackTimestamp` right below `tvTrackName` with a smaller font.
    * Update `TrackAdapter.java` to bind these new fields to the views.
