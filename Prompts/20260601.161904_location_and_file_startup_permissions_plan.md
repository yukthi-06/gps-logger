# Implementation Plan - Sequential Location & File Startup Permissions

We will revise the startup permission flow to guarantee that the standard Location (and Notification) permissions are requested immediately upon app startup, followed by the All Files (Storage) Access settings screen if needed.

## Solution Design

1. **Prioritize Runtime Permissions**:
   - In `requestPermissionsOnStartup()`, we will first check and request the standard runtime permissions (Location, Notifications, legacy Storage).
   - This ensures the location permission dialog is presented to the user immediately on launch without waiting for user action.

2. **Sequential File Access Redirection**:
   - If standard permissions are already granted on startup, we check and request `MANAGE_EXTERNAL_STORAGE`.
   - If they are NOT yet granted, we prompt the standard dialog.
   - When the user responds to the standard permission dialog (`onRequestPermissionsResult`), if standard permissions are granted and `isStartupPermissionRequest` is true, we immediately redirect the user to the All Files settings screen.

## Proposed Changes

### 1. Update `MainActivity.java`
- Modify `requestPermissionsOnStartup()` to check/request standard permissions first.
- Add `checkAndRequestFileAccessStartup()` to request external file storage access on API 30+.
- Update `onRequestPermissionsResult(...)` to invoke `checkAndRequestFileAccessStartup()` upon successful startup permission grant.
