# Implementation Plan for Map Style Loading Fallback (Offline Mode Support)

**User Prompt:**
> maps are downloading offline.
> But one map shows details, but other map is not showing details(the othe rmp used to show details before implementing offline downloads)

1. **Investigate the Problem**:
    * **Why it was happening**:
      * When we switched to the online `styleUrl` (`https://basemaps.cartocdn.com/...`), MapLibre must load the style document before it can invoke the `OnStyleLoaded` callback.
      * For Track A (which was downloaded offline using `OfflineManager`), the online style JSON is fully cached in the offline database (`offline.db`). Thus, even when offline, MapLibre retrieves the style from the local cache database, the style loads successfully, the callback fires, and the route/details are displayed.
      * For Track B (which was **not** downloaded offline), the online style JSON is not cached anywhere. Since the user is offline, MapLibre's request to fetch `styleUrl` fails. As a result, the `OnStyleLoaded` callback is never invoked, `loadTrackData()` is never called, and the map remains completely blank without even showing the red route polyline!
      * Before we switched to the online URL, we used a local `asset://osm_style.json` URI. Since this is local, it loaded instantly offline, allowing the route line to draw on a blank grid.
    * **The Solution**:
      * We will register MapLibre's `OnDidFailLoadingMapListener` on the `MapView`.
      * If the map fails to load the style (e.g. because we are offline and it's not cached), the listener will intercept the error, fall back to our local, JNI-safe `asset://osm_style.json` style, and invoke the route parsing/drawing code so the route polyline is still drawn perfectly.
      * If the user is online or the style is cached offline (Track A), the online style loads seamlessly, and the fallback is completely bypassed.

2. **Implementation Steps**:
    * **`MapActivity.java`**:
        * Declare a private boolean field `styleLoaded = false;` in the class body.
        * Inside `onMapReady()`:
          * Bind `mapView.addOnDidFailLoadingMapListener(...)`.
          * If a map loading failure triggers and `styleLoaded` is false, fall back to setting the style using `"asset://osm_style.json"`, then call `loadTrackData(...)` upon successful load.
          * Inside the primary `mapLibreMap.setStyle(...)` callback, set `styleLoaded = true` and call `loadTrackData(...)`.
