# Implementation Summary - Tracking Info Freeze Feature

Implemented a "Tick" mark next to the tracking info field to allow users to freeze the input once entered.

## Changes Made

### 1. New Asset: Checkmark Icon
Created `app/src/main/res/drawable/ic_check.xml` using a vector drawable to represent the "Tick" mark.

### 2. Layout Update: activity_main.xml
- Modified the tracking info card to wrap the `EditText` and the new `ImageView` (tick mark) in a horizontal `LinearLayout`.
- Set the `EditText` to `layout_weight="1"` to fill available space.
- Added the tick mark with a ripple background for better interactivity.

### 3. Logic Update: MainActivity.java
- Added initialization for the lock icon.
- Implemented `onLockTrackingInfoClicked()`:
    - Validates that text is not empty.
    - Disables the `EditText` (`setEnabled(false)`).
    - Disables the tick icon and reduces its alpha to 0.5 to show it's "locked".
- Updated `saveLocationDataToFile()` to re-enable the field and icon when the session is finished and the text is cleared, preparing it for the next session.
