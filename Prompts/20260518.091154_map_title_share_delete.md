# Implementation Plan for Dynamic Map Title, Sharing and Deletion

**User Prompt:**
> When user clicks on any item from Track List, the next screen shows the route on a map. The title of this screen should be the route name(list item text).
> Also this screen should have Delete and Share icons with respective functionality

1. **Investigate the Problem**:
    * Currently, `MapActivity` always sets its screen title to `"Track Map"` statically in `onCreate()`.
    * We should pass the track's dynamic display name from `TracksListActivity`'s list item click via the starting Intent (`TRACK_DISPLAY_NAME`), and use it to set `getSupportActionBar().setTitle(displayName)`.
    * The map screen does not have Share and Delete icons. We should:
      * Modify `res/menu/menu_map.xml` to include `action_share` (using standard `@android:drawable/ic_menu_share`) and `action_delete` (using `@android:drawable/ic_menu_delete`).
      * In `MapActivity.java`, handle these menu items inside `onOptionsItemSelected()`.
      * **Share implementation**: Detect all existing formats (`.kml`, `.gpx`, `.json`), pick the best one, construct its secure `FileProvider` URI, and trigger a native `Intent.ACTION_SEND` chooser.
      * **Delete implementation**: Display a secure `AlertDialog` confirmation. If confirmed, delete all three associated file formats, report success, and call `finish()` to return to the Track List screen (which automatically reloads in `onResume()`).

2. **Implementation Steps**:
    * **`TracksListActivity.java`**:
      * In `onMapClick()`, put the `displayName` extra: `intent.putExtra("TRACK_DISPLAY_NAME", trackItem.displayName);`.
    * **`res/menu/menu_map.xml`**:
      * Replace contents to include "Share", "Delete", and "Download Offline" items.
    * **`MapActivity.java`**:
      * Import `android.content.Intent`, `android.net.Uri`, `androidx.core.content.FileProvider`, and `android.app.AlertDialog`.
      * Read `TRACK_DISPLAY_NAME` extra in `onCreate()`, falling back to `baseName` if missing. Set it as the Action Bar/Toolbar title.
      * Modify `onOptionsItemSelected()` to handle `action_share` and `action_delete`.
      * Implement `shareTrack()` and `deleteTrack()` matching the exact logic used in `TracksListActivity.java`.
