# Implementation Plan - Fix File Writing Stoppage (Forbidden Colon Characters)

We will fix the issue where the app fails to write GPX, KML, and JSON files to disk.

## Root Cause Analysis
- In commit `ba278b2`, the timestamp formatting in `MainActivity.java` was changed from `yyyy-MM-dd.HHmmss` to `DateTimeFormatter.ISO_LOCAL_DATE_TIME`.
- `ISO_LOCAL_DATE_TIME` outputs standard ISO 8601 timestamps containing colons (like `2026-06-01T11:08:45`).
- Colons (`:`) are forbidden characters in standard Android and Windows filesystems (such as FAT32, exFAT, NTFS).
- Because the generated filenames contain colons, `java.io.FileOutputStream` throws an `IOException` / `FileNotFoundException` (Invalid argument) during `saveToFile`.
- As a result, the file writing fails completely, and no GPX, KML, or JSON files are saved to the device disk.

## Solution Design
- We will update the timestamp formatting in `MainActivity.java` to format using `ISO_LOCAL_DATE_TIME` and then replace all colons (`:`) with dashes (`-`) using `.replace(":", "-")`.
- This ensures the generated filename is fully safe for all Android filesystems.
- The `formatTimestamp` method in `TracksListActivity.java` already has a parser specifically written to replace dashes with colons back to standard display format (e.g. `2026-05-17T18-15-09 -> 2026-05-17 18:15:09`), meaning the list display will automatically render the timestamps correctly!

## Proposed Changes

### 1. Update `MainActivity.java` (in `saveSinglePointToFile` and `saveLocationDataToFile`)
- Change the `fileTimestamp` generation to replace colons with dashes:
  ```java
  String fileTimestamp = ZonedDateTime.now()
          .format(DateTimeFormatter.ISO_LOCAL_DATE_TIME)
          .replace(":", "-");
  ```
- Target file: `app/src/main/java/com/example/gpslocationlogger/MainActivity.java`
