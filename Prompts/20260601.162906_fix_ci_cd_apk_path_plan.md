Run mv app/build/outputs/apk/debug/app-debug.apk \

Error: Process completed with exit code 1.

# Implementation Plan - Fix CI/CD Pipeline APK Path Mismatch

We will resolve the CI/CD compilation pipeline error. The error occurred because introducing Gradle ABI splits causes the default debug target APK name to change from `app-debug.apk` to `app-universal-debug.apk` (and split ABI versions), breaking hardcoded scripts expecting `app-debug.apk`.

## Solution Design

We will write a standard dynamic Gradle variant renamer in `app/build.gradle`.
- We will iterate through `applicationVariants`.
- For the universal APK output (identified when the output's ABI filter is `null`), we will dynamically force the output filename:
  - For `debug` builds, rename it to exactly `app-debug.apk`.
  - For `release` builds, rename it to exactly `app-release.apk`.
- This ensures that the universal all-in-one APK is produced with the traditional `app-debug.apk` and `app-release.apk` names expected by CI/CD, while still building the optimized CPU-split APKs in parallel.

## Proposed Changes

### 1. Update `app/build.gradle`
- Insert the dynamic output renamer block in `android { ... }`:
```groovy
    applicationVariants.all { variant ->
        variant.outputs.all { output ->
            def abi = output.getFilter(com.android.build.OutputFile.FilterType.ABI)
            if (abi == null) {
                // Rename universal APK to standard name expected by CI/CD pipeline
                if (variant.buildType.name == "debug") {
                    outputFileName = "app-debug.apk"
                } else if (variant.buildType.name == "release") {
                    outputFileName = "app-release.apk"
                }
            }
        }
    }
```
