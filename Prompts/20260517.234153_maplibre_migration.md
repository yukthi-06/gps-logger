# Implementation Plan for Migrating to MapLibre SDK

**User Prompt:**
> Use the below library instead of osmdroid
>
>    // MapLibre SDK (Open Source fork of Mapbox)
>    implementation "org.maplibre.gl:android-sdk:11.5.0"
>    implementation "org.maplibre.gl:android-plugin-annotation-v9:3.0.1"

1. **Investigate the Problem**:
    * The user wants to switch the mapping library from `osmdroid` to `MapLibre SDK`.
    * This requires updating the `build.gradle` dependencies.
    * The layout file `activity_map.xml` must be updated to use `org.maplibre.android.maps.MapView`.
    * `MapActivity.java` needs a complete rewrite of the map initialization, layer setup, and polyline drawing logic using MapLibre's API and the annotation plugin.

2. **Implementation Steps**:
    * **`app/build.gradle`**:
        * Remove `osmdroid-android` and add the two `MapLibre` dependencies.
    * **`activity_map.xml`**:
        * Change `<org.osmdroid.views.MapView>` to `<org.maplibre.android.maps.MapView>`.
    * **`MapActivity.java`**:
        * Initialize `MapLibre.getInstance(this)`.
        * Handle map lifecycle methods (`onStart`, `onResume`, `onPause`, `onStop`, `onSaveInstanceState`, `onDestroy`, `onLowMemory`).
        * Call `mapView.getMapAsync` to get the `MapLibreMap` object.
        * Set a default raster style using OpenStreetMap tiles (via `Style.Builder().fromJson(...)`) so it works out of the box without an API key.
        * Parse the `.json` file to get a list of `com.mapbox.geojson.Point`s.
        * Use `LineManager` (from the annotation plugin) to draw the polyline.
        * Create a `LatLngBounds.Builder` with all points and use `mapLibreMap.easeCamera(CameraUpdateFactory.newLatLngBounds(bounds, padding))` to zoom into the route.
