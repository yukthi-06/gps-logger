# User Prompt
```
Add a button(above the three existing buttons) with label "Record a point".  The function of this button is to record a single GPS location(not a track).
Chose an appropiate format to save this info.  There is no Start/Stop for this.
```

## Overview
This implementation plan outlines the steps to add a new "Record a point" button to the main dashboard. This feature allows users to record a single GPS location immediately, saving it in the user's preferred formats (JSON, GPX, KML) without starting a continuous background tracking session.

## Implementation Steps

### 1. Update Layout (`activity_main.xml`)
- Add a new `Button` with ID `@+id/btnRecordPoint` and label `"Record a point"`.
- Position it directly below `@id/cardTrackingInfo` (which was the previous anchor for `@id/btnStartTracking`).
- Update `@id/btnStartTracking` to be positioned below `@id/btnRecordPoint`.
- Match the visual styles (colors, padding, corners) with the existing buttons.

### 2. Implement Location Fetching and Saving in `MainActivity.java`
- Import:
  - `com.google.android.gms.location.FusedLocationProviderClient`
  - `com.google.android.gms.location.LocationServices`
  - `com.google.android.gms.location.Priority`
- Initialize `FusedLocationProviderClient` in `onCreate`.
- Wire up the new button (`btnRecordPoint`) to check/request permissions and then query the current location.
- Use `getCurrentLocation(Priority.PRIORITY_HIGH_ACCURACY, null)` to query a fresh, high-accuracy GPS position.
- Implement a robust fallback to `getLastLocation()` in case `getCurrentLocation()` fails or returns null.
- Save the single point to files prefixed with `location_logs_` and containing `Recorded_Point` (e.g., `location_logs_[timestamp]_Recorded_Point_[notes].[ext]`) so that:
  - It matches the user's preferred formats (JSON, GPX, KML).
  - It integrates flawlessly with `TracksListActivity`'s parsing logic without breaking anything.
- Display a success `Toast` showing the captured coordinates and update `tvCoordinates` on the UI.

### 3. Improve Map Rendering for Single Points (`MapActivity.java`)
- Modify the GPX regex in `MapActivity.java` to support both `<trkpt>` and `<wpt>` tags:
  - From: `Pattern.compile("<trkpt lat=\"([^\"]+)\" lon=\"([^\"]+)\">")`
  - To: `Pattern.compile("<(trkpt|wpt) lat=\"([^\"]+)\" lon=\"([^\"]+)\">")`
  - Adjust group indices: Group 2 for latitude, Group 3 for longitude.
- Import `CircleManager` and `CircleOptions` from the MapLibre annotation library.
- In `loadTrackData()`, check if the number of points is exactly `1`.
  - If `1`, draw a highly-visible circle marker (`CircleOptions`) instead of a line.
  - If `> 1`, draw the polyline as before.
