# Implementation Plan - Advanced Point Sharing Options

This plan outlines the design and integration of advanced sharing/utility options specifically for single "Recorded Points" in the GPS Location Logger app.

## 1. Objectives & Requirements
When a user decides to share a recorded point, they should be presented with a menu of intuitive actions instead of immediately launching the system share sheet. The options include:
- **Show on Google Maps**: Launches a navigation/view intent in Google Maps or any other external map application showing the point.
- **Copy Google Maps URL**: Generates a standard Google Maps URL pointing to the coordinates and copies it to the clipboard.
- **Copy Lat/Lon values**: Formats the latitude and longitude coordinates and copies them to the clipboard (e.g. `9.931234, 76.261234`).
- **Share Location File(s)**: Invokes the existing share sheet flow, allowing users to share the physical GPX/KML/JSON/ZIP files.

## 2. Shared Helper Class (`SharePointHelper.java`)
To keep the codebase DRY and maintainable, a new helper utility `SharePointHelper.java` will be introduced. It will provide the following capabilities:
- **`tryExtractCoordinates(String baseName)`**: Parses the coordinates (latitude/longitude) of a recorded point from the saved GPX, KML, or JSON files.
- **`showShareOptionsDialog(Activity activity, String baseName, Runnable onShareFiles)`**: Shows an elegant `AlertDialog` displaying the options to the user and executing the selected actions.

### Elegant Options Dialog Layout
The dialog will present a list of choices:
1. `📍 Show on Google Maps`
2. `🔗 Copy Google Maps URL`
3. `📋 Copy Lat/Lon Values`
4. `📁 Share Location File(s)`

## 3. Integration Points

### A. Tracks List Activity (`TracksListActivity.java`)
- Update `onShareClick(TrackItem trackItem)`:
  - If `trackItem.isRecordedPoint()` is true, attempt to parse the coordinates.
  - If coordinates are successfully extracted, show the `SharePointHelper.showShareOptionsDialog` options dialog.
  - Otherwise, fallback to the existing file sharing flow.

### B. Map Activity (`MapActivity.java`)
- Update `shareTrack()`:
  - Check if `baseName != null && baseName.contains("_Recorded_Point")`.
  - If true, attempt to parse coordinates and display the options dialog.
  - Otherwise, fallback to the existing file sharing flow.

### C. Main Activity (`MainActivity.java`)
- Update `shareLastLog()`:
  - Check if `lastSavedName != null && lastSavedName.contains("_Recorded_Point")`.
  - If true, extract the base name (by stripping extension), attempt to parse coordinates, and show the options dialog.
  - Otherwise, fallback to the existing file sharing flow.

## 4. Verification Plan
- Record a point with a description.
- Go to the tracks list, tap the share icon for the recorded point. Verify that the new options dialog appears.
- Test "Show on Google Maps" and verify that it launches an external map app at the correct location.
- Test "Copy Google Maps URL" and verify that pasting outputs a valid Google Maps search URL.
- Test "Copy Lat/Lon values" and verify the clipboard has correct values formatted cleanly.
- Test "Share Location File(s)" and verify that it behaves identically to the previous version's share flow.
- Repeat verification from within `MapActivity` (using the Share button in the map view) and `MainActivity` (using the Share menu action for a freshly recorded point).
