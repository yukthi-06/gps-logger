# Implementation Plan for Clearing Offline Maps Cache

**User Prompt:**
> how to clear offline maps from this app

1. **Investigate the Problem**:
    * Currently, the app supports downloading offline map regions via a button on the Map screen.
    * There is no direct, user-friendly way to delete or clear these offline map regions within the application, short of clearing all App Data inside Android's system Settings (which resets all other options and preferences too).
    * We should implement a **"Clear Offline Maps"** button on the Settings screen.
    * Using MapLibre's `OfflineManager`, we will query all downloaded regions and delete them completely from the database. To prevent accidental deletion, we'll wrap the action in an AlertDialog confirmation modal.

2. **Implementation Steps**:
    * **`res/layout/activity_settings.xml`**:
        * Append a new section "OFFLINE STORAGE" at the bottom of the ScrollView.
        * Add a `CardView` containing a title, description, and a red styled `Button` (`@id/btnClearCache`).
    * **`SettingsActivity.java`**:
        * Import AlertDialog and AtomicInteger.
        * Bind the button in `onCreate`.
        * Implement `clearOfflineMapCache()` which gets `OfflineManager`, checks for regions, prompts the user with an AlertDialog, and systematically deletes all downloaded regions.
