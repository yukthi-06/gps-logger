# Implementation Plan - Ensure Minimum Map View Size of 20m x 20m

To meet the requirement that the starting map view is no less than 20 meters by 20 meters, we will implement bounds verification and expansion in `MapActivity.java`.

## Proposed Changes

### `MapActivity.java`

1. **Add `ensureMinBoundsSize` Utility Method**:
   Create a method `ensureMinBoundsSize(LatLngBounds bounds, double minSizeMeters)` which:
   - Takes a `LatLngBounds` and calculates its latitude/longitude spans.
   - Converts `20` meters to latitude degrees using the approximation `1 degree = 111,111 meters`.
   - Converts `20` meters to longitude degrees using `1 degree = 111,111 * cos(latitude) meters` at the center latitude of the bounds.
   - If either the latitude span or the longitude span is smaller than the required 20-meter delta, expands the bounds outward from the center coordinates.
   - Clamps the expanded bounds to valid geographic limits (`[-90, 90]` for latitude, `[-180, 180]` for longitude).
   - Returns a new adjusted `LatLngBounds`.

2. **Apply Bounds Expansion in `loadTrackData`**:
   - In `loadTrackData()`, after computing `trackBounds = boundsBuilder.build();`, apply our `ensureMinBoundsSize` logic to `trackBounds` with `minSizeMeters = 20.0`.
   - This ensures that:
     - The camera easing (`mapLibreMap.easeCamera(CameraUpdateFactory.newLatLngBounds(trackBounds, 100))`) uses bounds that are at least 20m x 20m.
     - The offline download logic (which relies on `trackBounds`) also uses the minimum bounds size, avoiding invalid / infinitely small tile requests for zero-length tracks.
