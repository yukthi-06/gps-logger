# Bug Fix - Missing tvSavePath Reference

Fixed a compilation error in `MainActivity.java` where the `tvSavePath` variable was being used but was not declared as a class field or initialized.

## Changes Made

### MainActivity.java
- Added `private TextView tvSavePath;` to the class-level UI fields.
- Initialized `tvSavePath` in `onCreate()` using `findViewById(R.id.tvSavePath)`.
- This ensures that the line `tvSavePath.setText(...)` in `saveLocationDataToFile()` has a valid reference to the UI element.

## Verification
- The app should now compile successfully.
- When a log is saved, the bottom label should correctly update with the filename.
